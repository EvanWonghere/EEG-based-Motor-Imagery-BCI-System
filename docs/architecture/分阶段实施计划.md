# 分阶段实施计划：从简陋版本到目标架构

> 基于 `项目架构设计.md` 的目标，将重构拆分为 6 个阶段
> 每个阶段结束后系统仍可正常运行（渐进式迁移，不破坏现有功能）

---

## 当前状态

```
python_backend/          # 扁平结构，~7 个文件
├── preprocessing.py     # 基本带通滤波 + 分段
├── training.py          # CSP+LDA/SVM/RF 流水线
├── train_model.py       # 入口脚本（500+ 行，数据加载+训练混杂）
├── datasets.py          # 数据集加载
├── analyze_model.py     # 模型分析
├── replay_stream.py     # LSL 回放
├── download_datasets.py # 数据下载
├── utils.py             # 常量与路径
└── test_datasets.py     # 数据集验证
```

**主要差距：**
- 无配置系统（参数硬编码）
- 无 CAR 重参考、ICA 伪迹去除、坏导检测
- 无 FBCSP（仅基础 CSP）
- 无 EEGNet 深度学习方法
- 无正式评估框架（无 Kappa、统计检验、跨方法对比）
- 可视化散落在各文件中
- Unity 前端仅基础接收

## 迁移策略

1. **并行共存：** 整个过程中 `python_backend/` 保持不动，新代码写入 `src/` + `scripts/`。直到所有阶段完成并验证后，再一次性移除旧代码。
2. **逐模块验证：** 每个阶段完成后，用对应的验证标准确认功能正确，再进入下一阶段。
3. **配置优先：** 从阶段零开始就用 YAML 配置驱动，后续所有模块都从配置中读取参数，不硬编码。
4. **测试覆盖：** 每个阶段附带对应的 `tests/` 测试文件，确保重构不引入回归。

---

## 建议优先级

如果时间紧张，核心路径为：

> **阶段零 → 阶段一 → 阶段二 → 阶段三a（FBCSP+SVM）→ 阶段四a（评估）→ 阶段四b（可视化）→ 阶段五（Web 在线仿真）**

这条路径覆盖了论文最核心的内容（第四~六章）+ 在线演示。阶段三b（EEGNet）和阶段六（Unity）可在核心路径完成后再补充。
